Real-Time Flight Bookability Predictor - Implementation Pipeline

Overview
This document outlines the detailed, step-by-step implementation pipeline of the Real-Time Flight Bookability Prediction system. It covers data ingestion, model training, evaluation, and the real-time inference application.

Phase 1: Data Ingestion & Preprocessing (process_full_data.py)
This phase transforms raw interaction logs into a machine-learning-ready dataset.

1. Datasets Overview
   - Input (Raw Data): `tbl_SearchTracking_Merged.csv`
     *Original Columns (subset)*: Id, SearchType, Origin, Destination, Airline, DepDate, ArrDate, LandingTime, PreviousPage, LandingPage, Class, Status
   - Output (Processed Data): `processed_flight_data_full.csv`
     *Resulting Columns*: trip_type, origin_airport, destination_airport, airline_code, cabin_class, meta_engine, days_to_departure, search_hour, search_day, dep_month, is_weekend, outcome_label

2. Memory Optimization
   - The raw CSV is extremely large. To prevent Out-Of-Memory (OOM) errors, it is processed iteratively using `chunksize=100000` via Pandas.

3. Feature Engineering & Preprocessing Pipeline
   The primary data transformation takes place in the `process_chunk()` function.

   Step 3.1: Temporal Feature Extraction
   We calculate the difference between the departure date and the search booking time.
   ```python
   chunk['departure_date'] = pd.to_datetime(chunk['DepDate'], errors='coerce')
   chunk['booking_time'] = pd.to_datetime(chunk['LandingTime'], errors='coerce')
   chunk['days_to_departure'] = (chunk['departure_date'] - chunk['booking_time']).dt.days

   # Granular temporal features
   chunk['search_hour'] = chunk['booking_time'].dt.hour
   chunk['search_day'] = chunk['booking_time'].dt.dayofweek
   chunk['dep_month'] = chunk['departure_date'].dt.month
   chunk['is_weekend'] = chunk['search_day'].isin([5, 6]).astype(int)
   ```

   Step 3.2: Contextual Extraction (Meta Engine)
   By parsing the `PreviousPage` URL referral string, we determine the acquisition channel (e.g. Skyscanner, Google).
   ```python
   def get_meta_engine(url):
       if pd.isna(url): return 'Direct'
       try:
           domain = urlparse(str(url)).netloc
           if 'skyscanner' in domain: return 'Skyscanner'
           elif 'google' in domain: return 'Google'
           elif 'kayak' in domain: return 'Kayak'
           elif 'momondo' in domain: return 'Momondo'
           elif 'carltonleisure' in domain: return 'Direct'
           else: return 'Other'
       except: return 'Other'

   chunk['meta_engine'] = chunk['PreviousPage'].apply(get_meta_engine)
   ```

   Step 3.3: Data Standardization & Cleansing
   Standardize column nomenclatures and neutralize missing values before model ingestion.
   ```python
   chunk = chunk.rename(columns={
       'SearchType': 'trip_type',
       'Origin': 'origin_airport',
       'Destination': 'destination_airport',
       'Airline': 'airline_code',
       'Class': 'cabin_class',
       'Status': 'outcome_label'
   })

   # Forward-fill categoricals
   chunk['airline_code'] = chunk['airline_code'].fillna('Unknown')
   chunk['cabin_class'] = chunk['cabin_class'].fillna('Unknown')
   
   # Drop erroneous sequences (e.g., negative days to depart)
   chunk = chunk[chunk['days_to_departure'] >= 0]
   ```

Phase 2: Model Training (train_production.py)
1. Data Preparation:
   - Load the `processed_flight_data_full.csv` file.
   - Encode the target variable (`outcome_label`) into numerical classes: Booked (0), Price Mismatch (1), Not Available (2), Not Booked (3).
   ```python
   outcome_map = {'Booked': 0, 'Price Mismatch': 1, 'Not Available': 2, 'Not Booked': 3}
   df['label_encoded'] = df['outcome_label'].map(outcome_map)
   ```
2. Advanced Feature Interaction:
   - Synthesize an additional categorical feature, `airline_origin`, combining the Airline Code and Origin Airport to allow the model to learn hub-specific booking trends.
   ```python
   X_train['airline_origin'] = X_train['airline_code'].astype(str) + "_" + X_train['origin_airport'].astype(str)
   X_test['airline_origin'] = X_test['airline_code'].astype(str) + "_" + X_test['origin_airport'].astype(str)
   ```
3. Cross-Validation:
   - Implement Stratified 5-Fold Cross-Validation to guarantee that rare classes (like Price Mismatch) are appropriately represented across all training splits.
4. Algorithm Initialization & Training:
   - Initialize and fit the `CatBoostClassifier` for MultiClass prediction.
   ```python
   model = CatBoostClassifier(
       iterations=2000,
       learning_rate=0.08,
       depth=8,
       loss_function='MultiClass',
       auto_class_weights='Balanced', 
       l2_leaf_reg=5,
       bagging_temperature=1,
       verbose=200,
       random_seed=42
   )
   model.fit(
       X_train, y_train,
       cat_features=current_cat_features,
       eval_set=(X_test, y_test),
       early_stopping_rounds=100,
       use_best_model=True
   )
   ```
5. Model Persistence:
   - Save the best performing iteration as the finalized binary artifact `catboost_production.cbm`.
   ```python
   final_model.save_model(MODEL_FILE)
   ```

Phase 3: Model Evaluation (evaluate_model.py)
1. Inference Execution: Load the finalized `catboost_production.cbm` model onto the processed dataset.
   ```python
   feature_order = base_categorical + target_encode_cols + numerical_features + ['airline_origin']
   X = df[feature_order]
   y = df['label_encoded']

   model = CatBoostClassifier()
   model.load_model('catboost_production.cbm')
   y_pred = model.predict(X)
   ```
2. Assessment:
   - Produce a detailed MultiClass Classification Report capturing Precision, Recall, and F1-Scores.
   - Build a Confusion Matrix to visually audit True versus False positive predictions.
   ```python
   print("\n--- Classification Report ---")
   print(classification_report(y, y_pred, target_names=outcome_map.keys()))

   print("\n--- Confusion Matrix ---")
   print(confusion_matrix(y, y_pred))
   ```

Phase 4: Real-Time Inference Application (app.py)
1. Application State: Launch Streamlit and load the serialized `catboost_production.cbm` model using `@st.cache_resource` for low-latency scoring.
   ```python
   @st.cache_resource
   def load_model():
       if not os.path.exists(MODEL_FILE):
           return None
       model = CatBoostClassifier()
       model.load_model(MODEL_FILE)
       return model

   model = load_model()
   ```
2. Real-Time Feature Transformation:
   - Dynamically calculate temporal requirements such as `days_to_departure` and `is_weekend` exclusively from user inputs.
   - Package all variables into a pandas DataFrame matching exact training schemas.
   ```python
   days_to_departure = (flight_date - search_date).days
   search_hour = search_time.hour
   search_day = search_date.day 
   dep_month = flight_date.month
   is_weekend = 1 if search_date.weekday() >= 5 else 0

   input_data = {
       'trip_type': trip_type, 'cabin_class': cabin_class, 'meta_engine': meta_engine,
       'origin_airport': origin, 'destination_airport': destination, 'airline_code': airline,
       'days_to_departure': float(days_to_departure), 'search_hour': float(search_hour),
       'search_day': float(search_day), 'dep_month': float(dep_month), 'is_weekend': int(is_weekend),
       'airline_origin': f"{airline}_{origin}"
   }
   input_df = pd.DataFrame([input_data])[feature_order] # Re-order exactly as training
   ```
3. Inference Generation & Feedback Delivery: Call `model.predict_proba()` against the user's data payload. Render UI results.
   ```python
   probs = model.predict_proba(input_df)[0]
   pred_class_idx = np.argmax(probs)
   
   outcome_map_reverse = {0: 'Booked', 1: 'Price Mismatch', 2: 'Not Available', 3: 'Not Booked'}
   predicted_label = outcome_map_reverse[pred_class_idx]
   ```
